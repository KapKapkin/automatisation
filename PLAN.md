## Этап 2: Проектирование базы данных

### 2.1 Модель "Корпус" (Building)
- [ ] id (PK)
- [ ] name — наименование корпуса
- [ ] address — адрес (опционально)
- [ ] description — описание

### 2.2 Модель "Расположение в корпусе" (Location)
- [ ] id (PK)
- [ ] building (FK → Building)
- [ ] name — название расположения (этаж, крыло и т.д.)
- [ ] ceiling_height — высота потолков в метрах

### 2.3 Модель "Подразделение" (Department)
- [ ] id (PK)
- [ ] name — название подразделения
- [ ] department_type — тип (факультет/кафедра/лаборатория)
- [ ] parent (FK → self, null) — родительское подразделение (иерархия)

### 2.4 Модель "Назначение помещения" (RoomPurpose)
- [ ] id (PK)
- [ ] name — назначение (аудитория, лаборатория, офис и т.д.)

### 2.5 Модель "Вид помещения" (RoomType)
- [ ] id (PK)
- [ ] name — вид (лекционная, компьютерный класс и т.д.)

### 2.6 Модель "Комната" (Room)
- [ ] id (PK)
- [ ] building (FK → Building)
- [ ] location (FK → Location)
- [ ] room_number — номер комнаты
- [ ] width — ширина в метрах
- [ ] length — длина в метрах
- [ ] purpose (FK → RoomPurpose)
- [ ] room_type (FK → RoomType)
- [ ] department (FK → Department)
- [ ] Свойства (вычисляемые):
  - area — площадь (width × length)
  - volume — объём (width × length × ceiling_height)

### 2.7 Миграции
- [ ] Создать миграции для всех моделей
- [ ] Применить миграции
